# Манифест совместной работы Antigravity (Manifest)

**Проект**: THz Unified Optimizer
**Цель**: Синхронизация работы агентов Antigravity между IDE и 2.0 (Chat) версиями, а также синхронизация с удаленным репозиторием GitHub.

## Описание архитектуры взаимодействия

Чтобы два разных экземпляра ИИ (или две разные сессии) могли понимать контекст друг друга, в проект внедрена следующая структура:

1. **`.agents/AGENTS.md` (Нативный файл настроек агентов)**
   - Это стандартный для архитектуры Antigravity способ задавать кастомные правила (Customization Rules) на уровне проекта.
   - Этот файл загружается агентами автоматически. Он содержит жесткие инструкции (проверять CHANGELOG, копировать артефакты, делать git push).

2. **`CHANGELOG.md`**
   - Служит "шиной сообщений" между сессиями. Каждая сессия оставляет след о том, какие файлы были изменены и какие решения приняты.

3. **`docs/artifacts/`**
   - Централизованное хранилище артефактов (планов, отчетов, задач). Обычно агенты сохраняют эти файлы глубоко в своих папках кэша (`~/.gemini/antigravity/brain/...`), но благодаря правилам из `AGENTS.md` они теперь будут дублировать их сюда, делая их видимыми и для IDE, и для чата, и для пользователя.

4. **Git и GitHub (Удаленная синхронизация)**
   - Для того чтобы код и контекст были доступны с разных устройств, агенты проинструктированы выполнять `git pull` в начале работы и делать осмысленные `git commit` и `git push` по завершении. Это гарантирует, что все артефакты и логи попадут в облако.

## Как использовать
При постановке задач агенту просто формулируйте их как обычно. Агент автоматически прочитает `.agents/AGENTS.md` и будет следовать заложенному пайплайну: 
`git pull` -> `Прочитать CHANGELOG` -> `Спланировать работу` -> `Написать код и сохранить артефакты` -> `Оставить запись в CHANGELOG` -> `git commit & push`.
