# План: Ablation Study (Исследование вклада компонент модели)

> **Для агента IDE**: Ваша задача — провести "Ablation Study" (пошаговое усложнение модели) для датасета `356att`, чтобы статистически доказать необходимость каждой физической поправки, и сгенерировать итоговый отчет и графики.

## 1. Методология исследования (Nested Models)
Вы должны написать скрипт `scripts/run_ablation_study.py`, который с помощью `lmfit` последовательно подгоняет 4 модели с постепенно возрастающей сложностью. Во всех моделях $P=15.5$ мкм жестко зафиксировано.

- **Model 0 (Pure Blanco)**:
  - Отключить модель Друде (закомментировать вызов `Z_surface_drude` или сделать флаг).
  - Отключить рассеяние: `loss_factor = 0` (vary=False).
  - Свободные параметры: `D_um`, `angle_offset`, `tau_ps`.
- **Model 1 (Blanco + Drude)**:
  - Включить импеданс Друде.
  - Рассеяние отключено: `loss_factor = 0` (vary=False).
  - Свободные параметры: `D_um`, `angle_offset`, `tau_ps`.
- **Model 2 (Drude + Scattering)**:
  - Включить Друде.
  - Включить рассеяние: `loss_factor` (свободен), `gamma` (фиксировано = 2.0).
- **Model 3 (Full Physics)**:
  - Друде + Рассеяние.
  - Все параметры свободны: `D_um`, `loss_factor`, `gamma`, `angle_offset`, `tau_ps`.

## 1.5 Анализ чувствительности (Post-Fit Ablation)
После получения оптимальных параметров для Model 3 (Full Physics), необходимо "заморозить" все параметры на их оптимальных значениях и по очереди отключать физические поправки, чтобы построить графики влияния каждой поправки на итоговую теоретическую кривую (при $\theta = 90^\circ$):
- Отключить `tau_ps` (принять $=0$).
- Отключить рассеяние (`loss_factor = 0`).
- Отключить модель Друде.
Этот шаг покажет, как именно каждая поправка визуально деформирует спектр и какие амплитудные/фазовые невязки она гасит.

## 2. Метрики для сбора
Для каждой модели скрипт должен извлечь из объекта результата `lmfit`:
1. `chi_square` ($\chi^2$)
2. `redchi` (Приведенный $\chi^2$)
3. `aic` (Akaike Information Criterion) — **Главная метрика выбора модели** (чем меньше, тем лучше. Падение AIC > 10 означает строгую необходимость введенного параметра).
4. `bic` (Bayesian Information Criterion)
5. RMSE невязок (корень из среднеквадратичного отклонения).

## 3. Графики для генерации
Скрипт должен создать и сохранить в `docs/images/` следующие графики:
1. `ablation_aic_comparison.png`: Столбчатая диаграмма (Bar chart) значений AIC для моделей 0, 1, 2, 3. Наглядно покажет обрушение ошибки.
2. `ablation_spectra.png`: Наложенные спектры пропускания при $\theta = 90^\circ$ (скрещенные). Эксперимент (точки) + 4 кривые от разных моделей. Обязательно сделать логарифмическую шкалу оси Y (dB) и приблизить (zoom) область 1.0 - 2.5 ТГц.
3. `ablation_residuals.png`: 4 сабплота (subplots) один под другим — графики остатков (Residuals vs Frequency) для каждой модели. Покажет, как исчезают "волны" и структура в шуме при усложнении модели.
