## ЧАСТЬ 1. Диагностика серий данных — причины аномалий

### 1.1 Корневая причина аномалии series1 (loss=113.94)

Изучение `data_pool/` выявило **критическую проблему с угловыми метками**:

| Серия | Угловые файлы | Ситуация |
|---|---|---|
| **356att** | −90°, −80°, ..., 0°, ..., +90° | ✅ 19 уникальных углов |
| **series1** | 0°–90° **и** 270°–350° | ⚠️ 270°→−90°, 280°→−80°... после нормализации — дубликаты! |
| **series2** | 0°–90° **и** 270°–350° | ⚠️ Аналогично series1 |
| **series3** | 0°–90° + 310°, 350° | ✅ 12 уникальных точек (310°→−50°, 350°→−10°) |
| **series4** | только 0° | ❌ 1 точка |
| **series5** | только 0° | ❌ 1 точка |

**Механизм аномалии**: `normalize_angle` переводит 270°→−90°, 280°→−80°. Оптимизатор получает **два измерения при одном и том же физическом угле**, снятых в разные сессии. Если между съёмками был дрейф источника → T(90°) ≠ T(−90°) → **внутреннее противоречие**. Оптимизатор не может это разрешить → loss=113.94.

### 1.2 Рекомендации по каждой серии

| Серия | Решение | Действие |
|---|---|---|
| **356att** | ✅ Основная | Использовать как есть |
| **series1** | ⚠️ Восстановить | Убрать файлы 270°–350° или проверить симметрию T(θ)=T(−θ) |
| **series2** | ⚠️ Восстановить | Аналогично series1 |
| **series3** | ✅ Лучшая | D=5.80 мкм (+2.3% от паспорта 5.67 мкм), 12 точек |
| **series4** | ❌ Исключить | 1 точка — задача неопределена математически |
| **series5** | ❌ Исключить | 1 точка — аналогично |
| **Global_Average** | ⚠️ Пересчитать | Только по 356att + series3 + series2 (0°–90°) |

Весь спектральный контент сводится к одному числу. P и D плохо обусловлены при таком подходе.

## ЧАСТЬ 3. Обзор физических моделей

### 3.1 Статья в docs/literature

**Castro-Camus E.** "Polarization-Resolved THz Time-Domain Spectroscopy"  
*J. Infrared Millimeter Terahertz Waves* **33**, 418–430 (2012). DOI: 10.1007/s10762-011-9856-8

Описывает формализм матриц Джонса для системы поляризаторов. Ваша реализация **полностью соответствует** этой теоретической базе. Модель Бланко используется для расчёта t_perp(ν) и t_par(ν) через эквивалентные импедансы (ряды Фурье).

### 3.2 Иерархия моделей

| Модель | Параметры | Что учитывает | Что упускает |
|---|---|---|---|
| Закон Малюса | 0 | Угловая зависимость | Всё |
| **Блэнко PEC** *(текущая)* | P, D | Геометрия, дифракция | σ металла, фаза, субстрат |
| **Блэнко + степ. закон** *(текущая)* | P, D, α, γ | + феноменол. потери | Физика потерь неверна |
| Morris et al. 2012 | η(ω), δ(ω) | + неполное подавление, фаза | Геометрический расчёт η |
| Whitbourn-Compton 1985 | P, D, n_sub | + диэлектрический субстрат | Конечная σ |
| **Блэнко + Друде** *(рекомендуется)* | P, D, σ₀, τ_D | + физически обоснованные потери/фаза | Субстрат, высшие порядки |
| RCWA + Друде | P, D, σ(ω), форма | Полный ЭМ расчёт | Медленный |

### 3.3 Почему Drude — следующий правильный шаг

Текущий степенной закон `exp(−αν^γ)` — это **подгонка кривой**, не физика:
- При series1 γ=3.0 (граница), при series2 γ=0.10 (граница) — оба вне физического смысла
- α и γ **коррелируют с D** → неединственное решение (отсюда расхождение 1D vs 2D на 25–30%)

**Модель Друде для вольфрама** (табличные значения, не подгоняются):
$$\sigma(\omega) = \frac{\sigma_0}{1 - i\omega\tau_D}, \quad \sigma_0 \approx 1.8\times10^7\ \text{(Ом·м)}^{-1},\ \tau_D \approx 8\ \text{фс}$$

Поверхностный импеданс:
$$Z_s(\omega) = \sqrt{\frac{i\omega\mu_0}{\sigma(\omega)}}$$

добавляется как поправка к Z_a, Z_b в модели Бланко. Параметры P и D становятся **единственными** свободными структурными параметрами.

### 3.4 Улучшение по Morris et al.

τ_ps сейчас — скалярная константа. Morris et al. показывают, что фазовый вклад **частотно-зависим**:
$$\varphi(\omega) = \varphi_0 + \varphi_1\omega + \varphi_2\omega^2$$
Добавление квадратичного члена (GVD) улучшает фазовую подгонку без новых физических параметров.

### 4.2 Критические пробелы

| Отсутствует | Почему важно |
|---|---|
| Траектория loss per iteration | Нельзя увидеть застревание |
| Промежуточные значения параметров | Нельзя видеть «куда уводит» D и γ |

| Доля 3σ-инлайеров | Нельзя оценить качество данных |

### 4.3 Минимальный callback для диагностики

```python
class ConvergenceLogger:
    def __init__(self, name, free_params, loss_fn, log_every=50):
        self.name, self.free_params = name, free_params
        self.loss_fn, self.log_every = loss_fn, log_every
        self.iter, self.best_loss = 0, np.inf
        self.history = []
    
    def __call__(self, xk):
        loss = self.loss_fn(xk)
        self.history.append(loss)
        if loss < self.best_loss: self.best_loss = loss
        if self.iter % self.log_every == 0:
            params = ', '.join(f"{n}={v:.4f}" for (n,_,_),v in zip(self.free_params,xk))
            logging.info(f"[{self.name}] iter={self.iter:4d} loss={loss:.6f} | {params}")
        self.iter += 1

# После minimize — проверить попадание на границы:
for val, (name, _, bounds) in zip(res.x, free_params):
    if abs(val - bounds[0]) < 1e-3 or abs(val - bounds[1]) < 1e-3:
        logging.warning(f"ПРЕДУПРЕЖДЕНИЕ: '{name}'={val:.4f} на границе {bounds}!")
```
