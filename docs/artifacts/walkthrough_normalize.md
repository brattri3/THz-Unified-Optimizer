# Результаты интеграции нормализации углов

Мы успешно интегрировали алгоритм нормализации углов в кодовую базу. Все углы теперь приводятся к физическому диапазону от -180° до 180°.

## Что было сделано

1. **Создание модуля утилит**:
   - Создан файл [utils.py](file:///c:/THz-Unified-Optimizer/unified_optimizer/utils.py) с функцией `normalize_angle(angle)`, которая сдвигает угол в диапазон [-180, 180] с помощью формулы `((angle + 180.0) % 360.0) - 180.0`.

2. **Интеграция в загрузчик данных**:
   - Функция `normalize_angle` интегрирована in [data_manager.py](file:///c:/THz-Unified-Optimizer/unified_optimizer/data_manager.py) внутри `parse_filename`. Это гарантирует, что при чтении любого файла данные автоматически привязываются к нормализованному углу.
   - Углы `270.0, 280.0, ..., 350.0` для датасета `series1` теперь автоматически мапятся на `-90.0, -80.0, ..., -10.0`.

3. **Автотесты**:
   - В [test_data_manager.py](file:///c:/THz-Unified-Optimizer/unified_optimizer/tests/test_data_manager.py) добавлены тестовые сценарии для проверки нормализации углов больше 180 градусов. Все тесты успешно пройдены (`Ran 3 tests in 0.002s, OK`).

4. **Правка совместимости Windows CLI**:
   - Из вывода скрипта `run_pipeline.py` удален символ ✅ (эмодзи), вызывавший ошибку кодировки `UnicodeEncodeError` в стандартном терминале Windows.
