# Правила для агентов Antigravity (Workspace Rules)

Этот проект редактируется одновременно через несколько интерфейсов Antigravity (Antigravity 2.0 Chat и Antigravity IDE). Для синхронизации контекста все агенты ДОЛЖНЫ строго соблюдать следующие правила:

## 1. Синхронизация через CHANGELOG
- **Чтение перед началом работы**: В начале любой новой задачи, прежде чем приступить к планированию или написанию кода, агент обязан прочитать файл `CHANGELOG.md` в корне проекта, чтобы узнать о последних изменениях, внесенных другими агентами.
- **Запись после завершения**: После выполнения задачи, внесения архитектурных изменений или завершения сессии, агент обязан добавить запись в конец файла `CHANGELOG.md`. 
  Запись должна содержать:
  - Дату и время (в локальном часовом поясе).
  - Название приложения/интерфейса (например, Antigravity IDE, Antigravity Chat).
  - Краткое описание проделанной работы.
  - (Опционально) Нерешенные проблемы или задачи для следующего агента.

## 2. Синхронизация с Git (GitHub)
- **Перед началом работы (Pull)**: Если проект инициализирован как git-репозиторий, перед началом внесения изменений агент должен выполнить `git status` и `git pull`, чтобы убедиться, что локальная копия содержит последние изменения с GitHub.
- **После завершения работы (Commit & Push)**: Завершив задачу, написав код, сохранив артефакты и обновив `CHANGELOG.md`, агент ОБЯЗАН:
  1. Добавить изменения: `git add .` (или добавить конкретные измененные файлы).
  2. Закоммитить с осмысленным сообщением на русском языке (ссылаясь на суть изменений, аналогично записи в CHANGELOG): `git commit -m "..."`.
  3. Отправить изменения в удаленный репозиторий: `git push`.
- **Конфликты**: Если при `git pull` или `git push` возникают конфликты, которые невозможно решить тривиально, агент должен остановиться и запросить помощь пользователя.

## 3. Сохранение артефактов в проекте
- По умолчанию агенты создают артефакты (планы, отчеты `walkthrough.md`, `task.md`) в своих изолированных директориях `<appDataDir>\brain\<conversation-id>`.
- **Копирование артефактов**: При создании или обновлении любого важного артефакта, агент ДОЛЖЕН создать его копию (или изначально сохранять) в папке `docs/artifacts/` внутри текущего проекта (`c:\THz-Unified-Optimizer\docs\artifacts\`).
- Это гарантирует, что другой агент из другого приложения сможет прочитать эти артефакты и понять ход мыслей или план. При копировании используйте понятные имена файлов, например: `docs/artifacts/plan_<topic>.md` или `docs/artifacts/walkthrough_task.md`.

## 4. Общие инструкции
- При редактировании кода старайтесь не ломать существующую логику, описанную в `README.md`.
- Сохраняйте русский язык для комментариев к коммитам, чейнджлога и документации, если не указано иное.

## 5. Стандарт статистического анализа и невязок
- **Обязательная оценка качества моделей**: При построении или модификации оптимизационных моделей (в том числе спектральных и интегральных) агент ДОЛЖЕН использовать методы статистического анализа невязок.
- **Проверка на нормальность**: Всегда проводите проверку распределения остатков на нормальность (визуально через Q-Q графики и/или статистическими тестами, такими как критерий Шапиро-Уилка или Харке-Бера). Наличие тяжелых хвостов (S-образная кривая) указывает на неучтенные выбросы.
- **Поиск скрытых паттернов**: Обязательно визуализируйте усредненные невязки (например, Residuals vs Frequency или Residuals vs Fitted values). Наличие волнообразных или структурных колебаний должно интерпретироваться как неполнота физической модели (например, неучтенная дифракция или спектр поглощения).
- **Документирование невязок**: Выводы о качестве аппроксимации должны подкрепляться 1D/2D картами остатков и сохраняться в аналитические отчеты.
