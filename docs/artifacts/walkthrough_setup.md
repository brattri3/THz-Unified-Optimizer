## Инструкция по проведению ручных экспериментов

Вы можете изменять исходный код алгоритмов, комментировать части кода и отслеживать, как это влияет на оптимизацию.

### 1. Как запускать тесты для проверки локальных изменений
Чтобы убедиться, что ваши правки не сломали базовую математику модели, запустите автотесты:
```powershell
.venv\Scripts\python -m unittest discover -s unified_optimizer/tests
```

### 2. Как запускать только нужную часть оптимизатора
С помощью CLI-аргументов вы можете запускать пайплайн точечно, чтобы не ждать долго:
* **Запустить только быстрый 1D-поиск для одного датасета `series1`**:
  ```powershell
  .venv\Scripts\python -m unified_optimizer.run_pipeline --dataset series1 --method 1d
  ```
* **Запустить только 2D-оптимизацию (Nelder-Mead) для `series2`**:
  ```powershell
  .venv\Scripts\python -m unified_optimizer.run_pipeline --dataset series2 --method 2d
  ```
* **Исключить долгий расчет глобального усреднения**:
  ```powershell
  .venv\Scripts\python -m unified_optimizer.run_pipeline --dataset series1 --no-global
  ```

## Как комментировать и отслеживать работу алгоритмов изнутри

### Эксперименты с 1D-оптимизацией ([optimizer_1d.py](file:///c:/THz-Unified-Optimizer/unified_optimizer/optimizer_1d.py))
Метод `optimize_1d_integral` состоит из 3 последовательных этапов минимизации ошибки:
1. **Этап 1 (строки 62-70)**: Оптимизация на линейной шкале.
2. **Этап 2 (строки 72-82)**: Оптимизация на логарифмической шкале (дБ) для уточнения параметров.
3. **Этап 3 (строки 84-94)**: Тонкая настройка с учетом сдвига угла решетки (`th_off`) и шумового порога (`eps`).

**Пример эксперимента**: Вы можете закомментировать Этап 2 и Этап 3, чтобы проверить, насколько точен поиск параметров только на линейной шкале. Для этого в конце функции `optimize_1d_integral` верните результаты первого этапа:
```python
    # Вместо результатов этапа 3 возвращаем результаты этапа 1
    p_f, d_f, a_f, g_f = res1.x
    return {
        'P_eff_um': p_f,
        'D_eff_um': d_f,
        'alpha': a_f,
        'gamma': g_f,
        'theta_offset': 0.0,
        'eps_floor': 0.0,
        'success': res1.success,
        'fun': res1.fun
    }
```

### Эксперименты с 2D-оптимизацией ([optimizer_2d.py](file:///c:/THz-Unified-Optimizer/unified_optimizer/optimizer_2d.py))
В функции `optimize_2d_spectral` минимизация идет методом `Nelder-Mead` (строки 118-123).
* Вы можете изменить веса `W_LINEAR` and `W_DB` в [config.py](file:///c:/THz-Unified-Optimizer/unified_optimizer/config.py), чтобы сместить приоритет оптимизатора в сторону линейного спектра или децибел.
* Вы можете поменять метод оптимизации, заменив `'Nelder-Mead'` на `'L-BFGS-B'` в строке 121 (но для этого потребуется добавить границы `bounds` в вызов `minimize`).
